还没学会写代码之前，我先学会了把东西做出来。

我不是技术背景。AI 最开始只是一个绕过限制才能用上的工具，后来慢慢变成一套方法：整理上下文、拆想法、反复改，最后把一个小东西真的上线。

真正的变化

AI 对我来说不再只是更快的搜索框，是从我意识到它能承接上下文开始的。

01

一开始不是策略，就是先想办法用上。

第一次知道 ChatGPT，注册本身就是个问题：IP 限制、账号限制，国内那套麻烦基本都遇上了。最后是靠计算机系同学帮忙，才把账号弄好。

刚开始用得很浅。写论文、问问题、查解释，比百度快，也更顺。但那时候我还没觉得它会改变什么。

ChatGPT搜索替代
02

NotebookLM 让我第一次看到“系统感”。

迪卡侬实习的时候，我意外拿到了一年 Gemini Pro。先是拿来写草稿、总结资料、中英文互译，都是很实用的工作场景。

后来发现 NotebookLM：把 PDF、文档、YouTube 链接丢进去，它只基于这些资料回答。做研究、写报告、整理信息的时候，这和普通聊天机器人完全不是一个感觉。它更像一个能工作的资料库。

Gemini ProNotebookLM基于资料回答

那是第一次，我觉得 AI 不是一个功能，而是一套系统。

Turning point
03

后来我开始把 Prompt 当成思考工具。

到了悉尼读研，我开始更刻意地用 ChatGPT。不只是要一个答案，而是观察什么会影响答案质量：上下文、限制条件、例子、语气，还有问题的顺序。

很多人会跳过这一层。我觉得这层很关键。问题说得越清楚，工具越有用。

Prompt 工程上下文设计
04

Claude 把它从聊天变成了执行。

我早就听说 Claude Code 在开发者里很火，但一直觉得跟我没关系。我不写代码，所以一开始只是把 Claude 当普通聊天助手用。

后来开始摸 Projects，发现长期上下文很有用。再到 Claude Code，之前一直躲，真正用起来才发现，门槛不是“我会不会写代码”，而是我能不能把想要的结果讲清楚，能不能看懂改了什么，能不能一轮轮修。

ClaudeClaude CodeProjects
05

第一个产品改变了问题本身。

第一个作品是个人记账工具。本来想做 App，Claude 建议我先做网页版：技术栈简单，能马上部署，手机电脑都能用。

我买了阿里云域名，部署上去了。界面很糙，但我一点都不在意。重点是，我一个人从想法做到上线，一个周末真的把东西做出来了。

在迪卡侬，我见过完整产品流程：BRD、PRD、开发迭代、测试、UAT、各方确认。那套流程很完整，但也很重。这次小很多、粗糙很多，也快很多。它让我觉得“做东西”这件事离我近了。

Vibe coding阿里云周末上线

我不是程序员，大概也不会马上变成程序员。但“我做不了东西”这个念头，已经不在了。

What changed
06

现在，我在搭自己的工作系统。

这个网站是我用 Claude Code 做的第二个东西。方法还是一样：讲清楚想要什么，看结果，指出哪里不对，继续改，然后上线。

我也在用 Obsidian 搭个人知识库。过去有太多想法和观察，因为没有好好记录，后来就散掉了。我不想继续这样。

目标不是一夜之间变成工程师，而是把这些工具理解到一定程度。等我真的有一个值得做的想法时，技术不要成为最先挡住它的东西。

个人网站Obsidian知识库

使用原则

我现在怎么用 AI

先给上下文

先把资料、背景和限制条件讲清楚，再要输出。上下文错了，答案再流畅也没用。

短循环迭代

不期待第一版完美。先做出来，再检查、指出问题、继续修。循环越短，结果越稳。

小东西先上线

不急着做宏大的产品。先把模糊想法变成真实可看的东西，这件事本身就很重要。
